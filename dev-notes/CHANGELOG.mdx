## 2026-05-09 — Phase 3 batch: 4 more atoms (status, kbd, avatar, dropdown)

Second parallel-agent batch lands the remaining atoms apart from
timeline (which was scoped out). Same orchestration pattern as the
first batch (worktrees + agents + central wiring PR):

- **`/components/status`** (PR #18) — `<Status />` indicator. 6
  semantic-palette buckets × 3 styles (dot, dot-pulse, bar) +
  optional `secondary` line.
- **`/components/kbd`** (PR #16) — `<Kbd />` and `<KbdCombo />`.
  Combos accept space-separated keys + optional description, with
  leading or trailing placement.
- **`/components/avatar`** (PR #19) — `<Avatar />` for users and
  orgs. Tinted-ink monogram, **no hue per user** (the design rule).
  4 sizes (20/26/32/48), 2 kinds (user radius 4 / org radius 2).
- **`/components/dropdown`** (PR #17) — `<Dropdown />` static-
  presentation surface. 4 states (rest/hover/focus/open). Real menu
  semantics deferred to a future PR.

Timeline atom was dropped from scope per user direction.

This PR (`chore/wire-atoms-batch-2`) flips the 4 nav todos to live,
registers 10 new TOC items across the 4 routes, and adds this
CHANGELOG entry.

## 2026-05-09 — Fix: zebra specimens + revert TopBar crumb

The Zebra-list specimens shipped by the parallel-agent batch diverged
from `staging/dreamlake-design-guide.html` §05 in three ways. Refactored
to match the source verbatim:

- **`ZebraSpecimen`** is now the all-in-one live specimen mirroring
  `#sub-zebra-live`: 8 rows with `rest`, `hover`, `selected`, and
  `focus` states inline (was: 6 plain zebra rows in isolation), with
  the rail-search header and the rail-demo box-shadow that the source
  CSS draws around `.rail-demo`.
- **`ZebraStatesSpecimen`** is now a state table — one row per band
  (base / zebra / hover-selectable / hover-warm / selected /
  focus-visible) with light + dark hex columns (was: a captioned three-
  row demo, which is the wrong specimen for this anchor).
- **`ZebraRunSpecimen`** now uses the ring-style selection from the
  source (transparent wrapper with `::after` accent border around the
  contiguous run; rows inside keep their zebra parity) instead of the
  fill-style alternative the agent picked. Updated `sections/multi-
  row.mdx` prose to describe the ring vs. fill choice and clarify
  that the specimen is the ring form.

Also reverted `src/components/TopBar.tsx` to the original static
`@dreamlake/uikit / Quick start` crumb. The route-aware lookup added
in PR #12 derived labels from `navGroups` (lowercase per design-guide
nav convention) which conflicted with the `<Crumbs>` casing each page
declares for itself. Drop the dynamic lookup; keep the original static
form. The page-level `<Crumbs>` (above each H1) remains the route-
specific breadcrumb.

## 2026-05-09 — Phase 3 batch: parallel agents land 4 pages

Four design-guide pages landed concurrently via subagents working in
isolated git worktrees, each on its own `feat/agent-*` branch:

- **`/geometry`** (PR #9) — Geometry: spacing scale, radii, hairlines,
  shadows, motion. New components: `<SpacingScale />`,
  `<RadiiSpecimen />`, `<HairlineSpecimen />`, `<ShadowSpecimen />`.
- **`/zebra-lists`** (PR #10) — The four-band model, per-state colors,
  multi-row runs. New components: `<ZebraSpecimen />`,
  `<ZebraStatesSpecimen />`, `<ZebraRunSpecimen />`. Multi-row run
  merging implemented in JS via `aria-selected` + `data-run-pos`.
- **`/components/chip`** (PR #11) — first reusable atom. Six tones
  (`neutral` / `accent` / `ok` / `warn` / `error` / `muted`) plus a
  leading-icon slot. The actual `<Chip />` lives at
  `src/components/Chip.tsx` (eventually exported from
  `@dreamlake/uikit`); `ChipSpecimen` under `src/components/design/`
  consumes it for the docs gallery.
- **`/components/button`** (PR #13) — second reusable atom. Two
  variants (`ghost`, `primary` — the only two the design-guide
  source demonstrates; no spec contamination from the original
  4-variant suggestion). `<Button />` at `src/components/Button.tsx`.

Orchestration pattern (worth keeping for future batches):
- Pre-create 4 git worktrees on `main` with branches `feat/agent-*`.
- Spawn 4 background `general-purpose` agents in parallel, each with
  a self-contained prompt that includes the source spec, file
  layout, and explicit "do NOT modify nav.ts / toc.ts / CHANGELOG.mdx"
  contract so the per-agent PRs never conflict on shared files.
- Merge agent PRs in order (no conflicts because of the contract).
- One central `chore/wire-agent-pages` PR adds the 4 nav entries
  (todos → live), 4 TOC route entries, and this CHANGELOG bullet
  — keeping the integration delta visible in one place.

This PR (`chore/wire-agent-pages`) is that consolidation.
