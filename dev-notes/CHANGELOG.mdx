## 2026-05-09 тАФ Consolidate: 8 foundation routes тЖТ one /style-guide page

Per-foundation routes (`/color`, `/typography`, `/spacing-layout`,
`/iconography`, `/elevation`, `/border-radius`, `/motion`,

## 2026-05-09 тАФ Quick fixes: lucide-react pin + typecheck in build

Two follow-ups from the recent reference-page batch:

- **`lucide-react` pinned to current 0.x** (`^0.577.0`). Was on the
  legacy `1.14.0` lineage that npm `latest` resolves to. The active
  Lucide release line is `0.x` (current stable: 0.577) тАФ that's what
  the design-guide spec calls for ("v0.460+"). The 1.14 line was
  abandoned years ago and lacks newer glyphs. IconCatalog imports
  unchanged; `pnpm build` clean.
- **`pnpm build` now runs `tsc --noEmit` first.** Type regressions
  block deploys instead of silently passing through. Resolves the
  out-of-scope follow-up flagged in the typecheck-sync changelog
  entry below.

Orchestration pattern (worth keeping for future batches):
- Pre-create 4 git worktrees on `main` with branches `feat/agent-*`.
- Spawn 4 background `general-purpose` agents in parallel, each with
  a self-contained prompt that includes the source spec, file
  layout, and explicit "do NOT modify nav.ts / toc.ts / CHANGELOG.mdx"
  contract so the per-agent PRs never conflict on shared files.
- Merge agent PRs in order (no conflicts because of the contract).
- One central `chore/wire-agent-pages` PR adds the 4 nav entries
  (todos тЖТ live), 4 TOC route entries, and this CHANGELOG bullet
  тАФ keeping the integration delta visible in one place.

This PR (`chore/wire-agent-pages`) is that consolidation.
