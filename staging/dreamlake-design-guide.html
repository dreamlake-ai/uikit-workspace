the surface contract
  1. Light separates --bg from --panel-bg; the rail shares --panel-bg.
  2. Dark collapses --panel-bg onto --bg; panels lift via 1px --faint + shadow, not a brighter fill.
  3. Dark drops the rail to #2b2b31 — figure/ground inverts; chrome recedes, working surface advances.
  4. Tokens.css is for surface chrome only — never --ok or --error. Keep semantic hex inline so grep tells the truth.

when adding a new concept
  1. Open pipelines-canvas.jsx. PIPE_KIND_COLOR, PIPE_STATUS, PIPE_FLOW_LABEL are canonical.
  2. Map by function, not vibe. "Ingest" is a successful inflow → green. "RC release" is active candidate → blue.

  4. grep '#[0-9a-f]\{6\}' across *.jsx should return only these six values plus surface tokens.

Zebra lists — the four-band model

selected > hover > zebra > base

Every list row is in exactly one band. Three different hues for three different jobs: zebra is structural (cool ink-step), hover is transient (selection-blue tint), selected is committed (saturated blue, white text).

six rules that matter most
  1. gap: 2px on the row container. Sacred.
  2. Backgrounds are solid hex, premixed off --bg. No rgba or runtime color-mix for state bgs.
  3. Zebra parity uses data-zebra from source array index — never :nth-child.
  4. Hover is bg-only. No inset border. The 2px gap already detaches the hovered tile.
  5. Text colors don't change between bands — except bright-blue selected forces #fff on every span.
  6. Multi-row run wrapper uses ::after with a 2px accent border, not an inset shadow.

        08⌗
        
Do · don't

      

      
If a design feels off, it's almost always one of these.

      

        

          
don't

          
    
            
  • Add a 7th hue to the semantic palette.
    • 
            
  • Promote semantic colors (--ok, --error) into tokens.css.
    • 
            
  • Use :nth-child(odd) for zebra parity in selectable lists.
    • 
            
  • Add an inset border on hover.
    • 
            
  • Lift the panel above --bg in dark mode.
    • 
            
  • Use a third font family or introduce a serif.
    • 
            
  • Animate box-shadow on row hover.
    • 
            
  • Use rgba() for state backgrounds.
    • 
          

        

        

          
do

          
    
            
  • Map new domain concepts to existing buckets by function.
    • 
            
  • Distinguish two same-bucket concepts with glyph + label.
    • 
            
  • Premix state backgrounds against --bg and ship the resulting hex.
    • 
            
  • Step row radius from 7px → 10px when selected.
    • 
            
  • Match the row gap (2px) on selection-run wrappers.
    • 
            
  • Layer multi-row ring as ::after, not inset shadow.
    • 
            
  • Force #fff on every span inside bright-blue selected rows.
    • 
            
  • Read --rail-bg; never inline-override per page.
    •

Icons — every glyph, all from Lucide

        ·
      

      

        Every icon across dreamlake, pipelines, timetravel, profile, and notebook is sourced from lucide.dev v0.460. We render them via window.lucide.icons[Name] so paths match upstream byte-for-byte — no hand-copied SVG. Stroke 1.4–1.6 at small sizes (12–16px); 1.1–1.25 at large (24–28px). Click any glyph to copy its kebab-case name.
