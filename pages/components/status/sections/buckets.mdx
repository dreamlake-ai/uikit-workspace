## Buckets

Six buckets, no seventh. Each pulls a single hex from the cross-page
semantic palette in `staging/design.md` — `active` `#23aaff`, `ok` `#1f8f4a`,
`model` `#7c5bd9`, `stale` `#c0922e`, `error` `#c8513b`, `idle` `#9c907a`.
Per the foundations rule, those hexes stay inline at the use site — there is
no `--ok` / `--error` token to read from. New domain concepts must slot into
an existing bucket; never invent a near-miss hex.
