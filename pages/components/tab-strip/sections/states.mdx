## Active states

The active tab is the only one painted ink; everything else holds at muted.
The 2px underline lives inside the active tab as an absolute-positioned
sibling, riding on the strip's bottom hairline at `bottom: -1px` so it
overpaints the border in ink wherever the active tab sits.

<TabStripStatesSpecimen />

These three cuts — first, middle, last — are how the static doc reads the
underline's behaviour: it tracks the active tab's width and offset, never
overhanging into the gap between tabs. The production component animates
the transition between active states using a FLIP-style measure-then-tween
on the underline rail; the docs deliberately render discrete frames so the
component reads correctly in the static pre-rendered HTML.
