## States

Four visual states: `rest`, `hover`, `focus`, `open`. Hover, focus, and open in the
specimen below are *forced* via the `state` prop — the styling is stamped
unconditionally so the row reads as a side-by-side reference, not as something you
have to mouse over or tab to.

<DropdownStatesSpecimen />

- **Rest.** Faint hairline border (`--color-faint`), panel-on-bg surface, mono 11.5px
  ink value, muted `▾` trailing glyph.
- **Hover.** Border lifts to `color-mix(in oklab, var(--color-ink) 18%, transparent)`.
  The trailing glyph and value text are unchanged — only the edge tightens.
- **Focus.** Same lifted border as hover, plus a 2px `--color-accent` outline at 50%
  alpha with 1px offset. Driven by `:focus-visible` in real use, so keyboard focus
  rings appear without polluting click-driven focus.
- **Open.** Same lifted border, with the `▾` chevron rotated 180° to read as `▴`.
  The popover menu itself is out of scope for this atom — the open state here is the
  visual cue for "menu is showing", nothing more.
