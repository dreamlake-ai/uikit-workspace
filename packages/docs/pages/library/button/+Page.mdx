# Button

The template's `library/` package ships with a single example
component to demonstrate the end-to-end authoring flow: a real
TypeScript source file, an example `.tsx` snippet, and an MDX page
(this one) that renders both. Use it as the pattern for every
component in your real library вЂ” see
**[How this page is wired](#how-this-page-is-wired)** at the bottom.

## Import

```tsx file="any-page.mdx"
import { Button } from '@dreamlake/uikit'
```

`'library'` is a Vite alias declared in `packages/docs/vite.config.ts`
that points at `packages/library/src/index.ts`. Rename the package
(and the alias key) to match your real library вЂ” see
**[Getting Started в†’ step 3](/getting-started#3-the-library-package--keep-rename-or-delete)**.

## Variants

Three visual weights: `primary` (accent fill), `secondary` (neutral
panel), `ghost` (transparent until hover).

Three files, one component. Reproduce this layout per component in

```text file="repo layout"
packages/

```

The page header imports the example twice вЂ” once as a default export
(for the live `<ButtonBasic />` render inside `<Preview>`) and once
with `?raw` (for the **Source** tab of `<Preview>`). The
`?raw` loader is enabled by a small Vite plugin in `vite.config.ts`:

```tsx file="packages/docs/pages/library/button/+Page.mdx"
import basicSrc from '../../../examples/Button.basic.tsx?raw'
import ButtonBasic from '../../../examples/Button.basic'

<Preview source={basicSrc} filename="Button.basic.tsx">
  <ButtonBasic />
</Preview>
```

That's the whole pattern. Add a new component:

1. Create `packages/library/src/components/Foo.tsx` and export it from
   `src/index.ts`.
2. Create `packages/docs/examples/Foo.basic.tsx` вЂ” a `default export`
   that returns the rendered demo.
3. Create `packages/docs/pages/library/foo/+Page.mdx` with frontmatter
   (`title: Foo`, `section: Library`, an `order`) and a `<Preview>`
   wrapping the example.

The sidebar picks up the new page automatically.
